# OpenBrain 0.1.0-alpha.2

This release ships engine fixes only. No vault migration is required; your files and configuration are untouched.

## Review after updating

1. Read the generated task in `MIGRATION_TODO.md` if one was created.
2. Run `open-brain scan` to regenerate local indexes. This release removes stale shard files automatically when a vault has shrunk below the shard threshold.
3. Run `open-brain health` to confirm the index is clean. A corrupt `vault.config.yml` is now reported as an error instead of silently ignored.

The update command replaces only the copied engine, the manifest, managed loader blocks, and this migration task file. It does not rewrite vault data or user-authored loader text outside OpenBrain markers.
